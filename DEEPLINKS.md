# WebVIZWallet — deep-links (предзаполнение форм через URL-hash)

Кошелёк умеет открывать нужную страницу и **предзаполнять поля формы** из URL. Это
позволяет давать пользователю готовую ссылку («перевести мне 10 VIZ», «проголосуй за
валидатора X», «купи аккаунт Y»), по которой форма уже заполнена — остаётся нажать кнопку.

## Как это устроено

Роутинг — по hash после `#`. Роутер (`change_state()` в `app.js`) разбирает:
- **сегменты пути**: `#/<view>/<subpage>/<arg>` → `path[1]`, `path[2]`, `path[3]…`;
- **query-параметры** после `?`: `#/<view>/<subpage>/?key=value&key2=value2` → `params.key`.

Каждый экран — функция `view_<name>(path, params, title)`; она читает `params.*`/`path[3]`
и пишет значения в поля формы. **Формы никогда не сабмитятся автоматически** — заполняется
только ввод, отправку делает пользователь кликом (безопасность: ссылка не может подписать
операцию за вас).

Значения query-параметров нужно **URL-кодировать** (пробел → `%20`, и т.д.).
Суммы — строкой с тикером, напр. `10.000%20VIZ`.

## Полный справочник

### Assets (активы)
| Действие | Ссылка | Параметры |
|---|---|---|
| Перевод токенов | `#/assets/transfer/?account=&amount=&memo=` | получатель, сумма, memo |
| Награда энергией | `#/assets/award/?to=&energy=&memo=` | аккаунт, энергия, memo |
| Фикс-награда | `#/assets/fixed-award/?to=&amount=&memo=` | аккаунт, сумма, memo |
| Делегировать доли | `#/assets/delegate-shares/?account=&amount=` | аккаунт, сумма |
| Застейкать доли | `#/assets/stake-shares/?amount=` | сумма |
| Расстейкать доли | `#/assets/unstake-shares/?amount=` | сумма |
| Инвайты/чеки | `#/assets/checks/?amount=&code=` | сумма создания, код для клейма |

### Account (аккаунты)
| Действие | Ссылка | Параметры |
|---|---|---|
| Создать аккаунт | `#/account/create-account/?login=&amount=&shares=` | логин, перевод, делегирование |
| Создать субаккаунт | `#/account/create-subaccount/?login=&amount=&shares=` | логин, перевод, делегирование |
| Купить аккаунт | `#/account/buy-account/<name>` | резолвит аккаунт и его цену |
| Купить short-аккаунт | `#/account/buy-short-account/<name>` | — |
| Купить субаккаунт | `#/account/buy-subaccount/<name>` | — |
| Продать аккаунт | `#/account/sell-account/?price=` | цена |
| Продать субаккаунт | `#/account/sell-subaccount/?login=&price=` | какой субаккаунт, цена |

### Market / подписки
| Действие | Ссылка | Параметры |
|---|---|---|
| Платные подписки провайдера | `#/market/paid-subscriptions/<provider>` | опции подписки |
| Создать платную подписку | `#/market/create-paid-subscribe/?descr=&url=&levels=&amount=&period=` | описание, url, уровни, цена, период |

### DAO / комитет / валидаторы
| Действие | Ссылка | Параметры |
|---|---|---|
| Заявка фонда (открыть) | `#/dao/fund-requests/<id>` | загрузка заявки по id |
| Создать заявку фонда | `#/dao/fund-create-request/?worker=&descr=&url=&min=&max=&duration=` | воркер, описание, url, min/max сумма, срок |
| Голос за валидатора | `#/dao/validators/?vote=<name>` | подсвечивает и прокручивает к валидатору (клик — за вами) |

### Prediction Markets
| Действие | Ссылка | Параметры |
|---|---|---|
| Открыть рынок | `#/pm/<market_id>` | загрузка карточки |
| Рынок + готовая ставка | `#/pm/<market_id>?amount=&side=` | сумма + подсветка стороны (binary: `side=0/1`; multi: `oindex=<n>`) |

### Служебное
| Действие | Ссылка | Параметры |
|---|---|---|
| Логин с возвратом | `#/login/?back=<path>` | куда вернуться после входа |
| Memo-ключ с возвратом | `#/memo/?back=<path>` | — |
| Пагинация списков | `…?page=<n>` | buy-account / buy-subaccount / подписки |

## Примеры

- Попросить перевод: `#/assets/transfer/?account=alice&amount=10.000%20VIZ&memo=for%20coffee`
- Онбординг-ссылка на создание аккаунта: `#/account/create-account/?login=newbie&amount=1.000%20VIZ`
- Кампания валидатора: `#/dao/validators/?vote=on1x`
- Прямая ссылка на рынок со ставкой: `#/pm/10?amount=5.000%20VIZ&side=0`
- Продать аккаунт по фикс-цене: `#/account/sell-account/?price=1000.000%20VIZ`

## Примечания
- `#/assets/deposit` (клейм-чек) — обработчик в текущей сборке отключён (закомментирован),
  поэтому deep-link для него не добавлен. Включить при реактивации страницы (`?code=`).
- Настройки (NS/Profile/Access/Security) грузятся из метаданных аккаунта — deep-link не применяется.
- Все суммы принимаются в том же формате, что и в форме (`N.NNN VIZ`).
